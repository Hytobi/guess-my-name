# Schéma données (référence — évolution vers Firebase)

Ce fichier décrit la structure prévue pour la base (Firestore + Storage pour les fichiers).
Il sera mis à jour au fil du projet.

## Modes applicatifs (impl. `src/lib/dataMode.ts`)
- **Développement** (`vite` / `import.meta.env.DEV`) : données dans **localStorage** (comportement historique).
- **Production** (`vite build` / `import.meta.env.PROD`) : **Firestore** (collections ci-dessous) + `onSnapshot` pour le rafraîchissement.
- Forcer Firestore en local : `VITE_USE_FIREBASE=true`. Forcer le local en preview prod : `VITE_USE_FIREBASE=false`.

## Firestore (collections)
- `users` — document id = `userid`, champs : `name`, `codeconnexion`
- `enigmes` — document id = `enigmeid`, champs alignés sur la table enigme (+ `imageDataUrl` optionnel)
- `guesses` — document id = `guesslistid`, champs alignés sur guessList

Règles de sécurité : voir `firestore.rules` (MVP ouvert ; à durcir avec Firebase Auth).

## Firebase Storage (`storage.rules`)
- Chemin prévu pour les images d’énigmes : `enigmes/{enigmeId}/{fileName}`.
- **Lecture** : publique (`allow read: if true`).
- **Écriture** : uniquement si l’utilisateur est connecté avec **Firebase Auth** et que le document Firestore **`admins/{uid}`** (même `uid` que `request.auth.uid`) contient **`enabled: true`**. Les documents `admins/*` ne sont pas modifiables par le client (règles Firestore `allow false`) : création manuelle dans la console une fois l’UID admin connu.
- La page admin appelle `signInWithEmailAndPassword` avec `VITE_FIREBASE_ADMIN_EMAIL` / `VITE_FIREBASE_ADMIN_PASSWORD` après le mot de passe applicatif.

## user
- userid : identifiant unique utilisateur
- name : nom affiché / saisi
- codeconnexion : code numérique à 8 chiffres, généré à l’enregistrement du nom ; permet de se reconnecter (autre appareil une fois la base centralisée, ex. Firebase)

## enigme
- enigmeid : identifiant unique
- libelle : titre / libellé
- date : jour de référence (AAAA-MM-JJ, calendrier local)
- nomFichier : nom du fichier image (côté Storage après migration Firebase)
- message : texte associé à l’énigme

Règle d’affichage sur l’accueil joueur : lister les énigmes dont le jour `date` est strictement antérieur au jour courant (implémenté dans `src/lib/dates.ts`).

## guessList
- guesslistid : identifiant de la ligne
- weeknumber : semaine ISO encodée (année × 100 + numéro de semaine, ex. 202615)
- guess : proposition de l’utilisateur pour la semaine en cours (modifiable à tout moment)
- enigmeid : énigme concernée
- userid : auteur de la proposition
- userName (optionnel, dénormalisé) : copie du nom affiché au moment de l’enregistrement ; en production on pourra joindre `user.name` à la place
